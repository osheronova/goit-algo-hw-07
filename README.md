# goit-algo-hw-07 — Дерева та балансування

Домашнє завдання до теми **“Дерева та балансування”**.

У роботі використана одна реалізація двійкового дерева пошуку (BST), поверх якої реалізовані три окремі функції для пошуку мінімального, максимального значення та суми всіх елементів дерева. Це дозволяє уникнути дублювання коду та сфокусуватися саме на алгоритмах.

---

## Структура проєкту

- `task_1_3.py`  
  Реалізація BST та функцій для завдань 1–3:
  - `find_max(root)` — знаходить найбільше значення в дереві (Завдання 1)
  - `find_min(root)` — знаходить найменше значення в дереві (Завдання 2)
  - `sum_tree(root)` — повертає суму всіх значень у дереві (Завдання 3)

  У блоці `if __name__ == "__main__":` є простий приклад створення дерева, вставки елементів та виклику всіх трьох функцій.

- `task_4.py`  
Додаткове завдання: можливо буде реалызовано пізніше, після завершення основних завдань і дотримання дедлайнів, що стає дуже складною задачею перед Різдвяними святами---

## Чому завдання 1–3 об’єднані в один файл

В умовах ДЗ немає вимоги розносити завдання 1–3 в окремі файли. Натомість є вимоги:

- реалізувати функцію для пошуку **максимального** значення в дереві,
- реалізувати функцію для пошуку **мінімального** значення в дереві,
- реалізувати функцію для обчислення **суми всіх значень** в дереві.

Усі ці функції працюють поверх однієї і тієї ж структури даних — двійкового дерева пошуку.  
Щоб не дублювати код класу `Node` та логіку вставки елементів, завдання 1–3 реалізовані в одному модулі:

- одна реалізація BST,
- три незалежні функції-алгоритми:
  - `find_max`
  - `find_min`
  - `sum_tree`

Такий підхід зменшує дублювання коду (DRY) і робить проєкт простішим для підтримки, не порушуючи вимоги завдання.
